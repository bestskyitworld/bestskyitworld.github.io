什么是MyBatis

MyBatis 是一款优秀的持久层框架，它支持自定义 SQL、存储过程以及高级映射。MyBatis免除了几乎所有的JDBC代码以及设置参数和获取结果集的工作。MyBatis 可以通过简单的 XML 或注解来配置和映射原始类型、接口和 Java POJO（Plain Old Java Objects，普通老式 Java 对象）为数据库中的记录。

这是官网的原话，简单来讲就是简化原始的JDBC操作，为我们数据库的增删改查提供更为方便的操作。下面将介绍第一个MyBatis程序的创建和介绍。

创建第一个MyBatis程序

Step1

首先创建一个最普通的Maven项目，就是不利用模板的Maven项目。

创建完项目后，我们删掉项目原始的src目录，这样做的目的是为了让整个项目成为一个父工程，之后的所有子项目通过创建的一个新的模块来进行添加，这样便于我们后续项目拓展。当然这只是一个个人习惯，你也可以不用这么做。

Step2

创建完子工程之后，我们需要做的就是导入MyBatis的依赖，用过Maven的小伙伴肯定再熟悉不过了。我们一共需要三个jar包，第一个是MySql数据库的驱动，没有这个就没有Mybatis，第二个就是Mybatis的依赖，第三个包是junit测试包，便于我们对程序进行测试。

这些jar包我们是在父工程的pom.xml中导入的，所以子工程不再需要引入这些jar包，第一步的好处就体现出来了，以后子工程单独需要的依赖我们单独导入就行，父工程最基本的依赖所有项目都能用，不用在子工程中一一导入。

Step3

第三步就是我们核心部分，前两步都是为我们写代码而搭建环境。
